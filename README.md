# Airtel — Icertis CLM Platform

Two tools in one serverless Next.js app, deployed free on Vercel:

1. **SOW Form** — Submit Statements of Work with mandatory Claude AI validation
2. **Icertis AI Chatbot** — RAG-based support chatbot for the Icertis CLM platform

---

## Features

### RAG Chatbot (new)
- Upload PDF / DOCX / TXT / Markdown documents as a knowledge base
- Embeddings generated via HuggingFace Inference API (free tier)
- Semantic search via Supabase **pgvector** (already in project)
- Answers generated by **Groq LLaMA 3.1 8B** (free, ~400 tok/s)
- Streaming responses with source citations
- Suggested questions, conversation history

### SOW Form (existing)
- Dynamic form with all standard procurement fields
- Claude AI validation before submission
- Field-level inline feedback (green/yellow/red borders)
- Supabase storage

## Setup

### 1. Clone & Install

```bash
npm install
```

### 2. Set up Supabase

1. Create a project at [supabase.com](https://supabase.com)
2. In the Supabase SQL editor, run **both** SQL files:
   - `supabase/schema.sql` (SOW form tables)
   - `supabase/rag-schema.sql` (RAG tables + pgvector)
3. Copy your project URL, anon key, and service role key

### 3. Get free API keys

| Service | URL | Notes |
|---------|-----|-------|
| Groq | [console.groq.com](https://console.groq.com) | No credit card |
| HuggingFace | [huggingface.co/settings/tokens](https://huggingface.co/settings/tokens) | Read token, no CC |
| Anthropic | [console.anthropic.com](https://console.anthropic.com) | SOW form only |

### 4. Configure Environment Variables

```bash
cp .env.local.example .env.local
# Fill in your keys
```

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
ANTHROPIC_API_KEY=your-anthropic-api-key
GROQ_API_KEY=your-groq-api-key
HUGGINGFACE_API_KEY=your-huggingface-read-token
```

### 5. Run Locally

```bash
npm run dev
```

Navigate to:
- `http://localhost:3000` — SOW Form
- `http://localhost:3000/upload` — Upload documents to knowledge base
- `http://localhost:3000/chat` — Icertis AI chatbot

## Deploy to Vercel

```bash
npm i -g vercel
vercel

# Add all env vars:
vercel env add NEXT_PUBLIC_SUPABASE_URL
vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY
vercel env add SUPABASE_SERVICE_ROLE_KEY
vercel env add ANTHROPIC_API_KEY
vercel env add GROQ_API_KEY
vercel env add HUGGINGFACE_API_KEY
```

Or connect your GitHub repo to Vercel for automatic deployments.

> **Note on free-tier limits:** Vercel Hobby has a 10 s serverless timeout. Large documents (> 5 MB) may exceed this during embedding generation. For larger files, upgrade to Vercel Pro (60 s timeout) or split documents into smaller parts.

## How the RAG Chatbot Works

```
User question
    │
    ▼
Generate query embedding  ──── HuggingFace all-MiniLM-L6-v2 (384-dim)
    │
    ▼
Vector similarity search  ──── Supabase pgvector  (cosine, top-5 chunks)
    │
    ▼
Build prompt with context ──── System prompt + retrieved chunks + history
    │
    ▼
Stream LLM response       ──── Groq llama-3.1-8b-instant
    │
    ▼
Return with source citations
```

### Document Upload Flow

1. User uploads PDF / DOCX / TXT / MD (max 10 MB)
2. Text extracted: `pdf-parse` (PDF), `mammoth` (DOCX), raw read (TXT/MD)
3. Text split into ~1 000-char overlapping chunks
4. Embeddings generated in batches via HuggingFace API
5. Chunks + embeddings stored in `rag_chunks` table (Supabase pgvector)

## How SOW AI Validation Works

1. User fills in the SOW form
2. Clicks **Run AI Validation**
3. Claude reviews all fields for completeness, logic, and business sense
4. Returns a score (0–100), blockers, and field-level warnings
5. If score ≥ 70 with zero blockers → **Submit SOW** button unlocks
